# 技术文档判官 Skill Spec

## Why
当前工作区缺少一个专门面向技术文档评审的 Skill，导致对技术文档的批判性反馈、优点归纳和可执行改进建议缺少统一标准。
该 Skill 需要帮助用户系统化审视文档质量，既指出问题，也保留做得好的部分，并给出足够细化的优化方案。

## What Changes
- 新增一个用于评审技术文档的自定义 Skill 规格
- 定义 Skill 的输入范围，包括技术方案、设计文档、原理笔记、架构说明等文档内容
- 定义 Skill 的输出结构，包括问题批判、亮点总结、详细改进方案
- 约束 Skill 的评审风格为“严格但建设性”，避免只给泛泛评价
- 规定 Skill 在用户要求“评审文档”“批判文档”“指出改进点”等场景下触发

## Impact
- Affected specs: 自定义 Skill 创建流程、技术文档评审能力
- Affected code: `.trae/skills/<skill-name>/SKILL.md`

## ADDED Requirements
### Requirement: 技术文档批判性评审
系统 SHALL 提供一个用于评审技术文档的 Skill，能够针对用户提供的技术文档内容输出结构化批判意见、亮点总结和详细改进方案。

#### Scenario: 识别文档问题
- **WHEN** 用户请求对一篇技术文档进行批判性评审
- **THEN** Skill 输出文档中的主要问题，优先指出结构混乱、术语不准、论证不足、信息缺失、可读性差等关键缺陷

#### Scenario: 归纳文档亮点
- **WHEN** 文档中存在表达清晰、结构合理、示例充分或术语准确等优点
- **THEN** Skill 明确指出这些优点，避免只给负面评价

#### Scenario: 提供可执行改进方案
- **WHEN** Skill 发现文档存在问题
- **THEN** Skill 给出可执行且足够具体的改进建议，包括建议改写方向、建议补充的信息类型、建议调整的章节结构或表达方式

### Requirement: 明确 Skill 触发条件
系统 SHALL 在用户明确要求评审、批判、改进技术文档，或上下文清楚表明目标是优化技术文档质量时使用该 Skill。

#### Scenario: 用户明确要求评审文档
- **WHEN** 用户提出“帮我批判这篇技术文档”或类似请求
- **THEN** 系统应识别并使用该 Skill

#### Scenario: 用户未要求文档评审
- **WHEN** 用户请求与技术文档评审无关
- **THEN** 系统不应误用该 Skill
