LLM-Learning 记录

前言：在 LLM 淘汰我这个搬砖仔之前，决定死的瞑目些，于是开始了对 LLM 的了解。

Intro to Large Language Models 🔗

推荐这个视频：https://www.youtube.com/watch?v=zjkBMFhNj_g

Key Insights 🔗

  1. LLM 的本质是数据压缩，模仿已有数据的行为。但不负责事实，不具备推理能力。（很好，我还能干到 35 岁

  1. 得到 GPT 一般分为两步：预训练和微调。由于模型很大，直接 train-from-scratch 很不划算，一般都是会在基座模型的基础上基于更新的数据加工。通常得到 GPT 之后还会有 RAG 以及一些 prompt engineering 来得到满意的回答。（有的时候你也会引导 GPT 得到自己想要的回答，某种程度就是 prompt engineering

  1. LLM 现在也面临很多挑战，比如 jailbreak attack (越狱攻击), 绕过安全措施的问答； prompt injection attacks（提示注入），导致回答结果带有不良信息，比如诈骗网站之类的，data posioning / backdoor attacks（数据操纵/后门攻击） ，比如带有一些恶意触发的 prompt 让 GPT 泄漏用户隐私。

  2. LLM 未来的展望。

    大部分都是从产品的角度上来展望的。

    • 简单点就是接着执行 scaling-law，大力出奇迹；
    • 搜索引擎，比如现在大家都在用的 perplexity；
    • 交互式编程，比如对于一些可以直接 fetch 的信息，自动出图，拟合等等；现在还想还没看来类似的产品出现, 还是需要人工闭环的。
    • 多模态，比如 Vision-Language-Model，或者短视频大热的 chatGPT 的 DAN 模式，赛博男友。目前都是的模态融合似乎还是很有探索的空间。
    • 推理，人类的思维模式是有快慢系统的，快系统就是很多知识缓存在了脑子，慢系统则需要一些时间计算；比如我们可以很快得到 10+12=22 这样的计算，但是要计算 127*123+231 就得慢慢算了；LLM 的只是做了一部分的快系统，而需要推理能力，更长时间计算的慢系统则是需要更多的时间。
    • 如何超越数据本身，其实和推理有一点类似；但还有差别，所以单独拎出来讲；所谓的超越就比如 AlphaGO，给他的数据都是人类的数据，但是他为什么可以在下棋的时候超越人类呢？因为围棋可以用模拟器来模拟之后发生的事情，得到 rewards，找到当前最佳的选择。但是对于 LLM 来说这种 rewards 是很难定义的，这是一个更开放的空间；目前还没有看到类似的进展。
    • LLM 作为系统的一部分；我觉得这是最兴奋的一个点；LLM 作为硬件基础服务的一部分，嵌入日常的生活，大佬给了一张图介绍 LLMOS的架构；同时我觉得这可能是下一代的硬件形态。

  3. 总结 & 思考：LLM 的浪潮真的很猛，虽然提到一些缺陷，但我觉得都是可以被解决的；更值得被思考的时，未来我们和 LLM 的关系，LLM 可以再怎么加入我们的生活；就我个人而说，已经离不开 GPT，视频课代表，monica.im 这些工具了；但是在更工业的方向，LLM 可以做什么呢？比如和机器人的结合，比如在自动驾驶领域等等，期待 LLM 干掉我的那一天，我也就再也不用焦虑失业，而是踏踏实实的躺平。（bushi
