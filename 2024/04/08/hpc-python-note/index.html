Chapter 1: 理解 Python 🔗

C1 主要介绍了计算机的存储结构 (RAM, L1/L2 Cache)，通信层 （总线）, python的特点。比较重要的信息如下：

  1. 由于全局解释器锁（GIL）的存在，每个CPU核心一次只能执行一个线程的Python字节码。对于CPU密集型任务，多线程通常无法提升性能。然而，对于IO密集型任务，多线程可以有效提升性能。因为在等待IO完成时，Python线程会释放GIL，允许其他线程执行，从而充分利用了CPU资源。

  2. Python由于其动态特性，无法像C++那样直接利用向量化加速。但是可以通过第三方库，如NumPy、Cython、OpenCV等，来实现类似的功能。

  3. GPU和CPU之间通过PCI总线进行通信，相较于前端总线，PCI总线的传输速度较慢。因此，GPU的数据传输通常是一个耗时的操作。

  4. Python对象的内存布局并非最优，这是由于Python是一种垃圾回收语言，内存会自动分配和释放。这可能导致内存碎片化，影响数据传输到CPU缓存的效率。

Chapter 2: 分析性能瓶颈 🔗

C2 主要讲了一些 profile的方法，并利用这些工具来找到耗时瓶颈。主要方法如下：

  1. timeit：一个Python模块，可以用于测量小段代码的执行时间。 e.g: python -m timeit -n 5 -r 5 -s "import julia1" "julia1.calc_pure_python(desired_width=1000, max_iterations=300)"

  2. Unix time命令：一个Unix系统命令，可以用来测量程序的执行时间。e.g. /usr/bin/time -p python julia1_nopil.py

  3. cProfile: Python标准库中的性能分析工具, 测量代码中各个函数的执行时间和调用次数。 e.g.

  1. line_profiler：用于进行逐行性能分析的工具，可以提供每一行代码的执行次数和消耗时间. 逐行分析会产生较大的性能开销 e.g.
  1. memory_profiler：用于分析内存使用情况的工具，可以帮助识别内存泄漏和优化内存使用。e.g.
  1. heapy: 一个用于分析 Python 堆对象的库，它可以帮助你追踪内存中的所有对象，并提供详细的分析报告，包括每种对象类型的数量、大小、引用关系等信息。

  1. dis: 用于反汇编Python字节码的工具，可以帮助理解Python代码背后的CPython虚拟机指令。 import dis; dis.dis(my_function). 如果我们的字节码指令更少，通常意味着在运行时虚拟机需要执行的操作也较少。

  2. Dowser: 一个用于分析 Python 程序内存使用情况的工具，它提供了一个简单易用的 Web 界面，用于展示 Python 程序的内存占用情况.

  1. perf: 一个强大的 Linux 性能计数器工具，可以用于分析系统级的性能指标，如 CPU 使用率、内存访问、磁盘 I/O 等
